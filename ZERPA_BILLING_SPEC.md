# ZERPA — Billing Module Spec
## `zerpa/billing`

---

## Overview

The Billing module is a top-level sidebar item. It expands into four sub-pages:

```
Sidebar → Billing
              ├── Quotes
              ├── Invoices
              ├── Products & Services
              └── Automated Invoices
```

All four sections are interconnected:
- A Quote can be converted to an Invoice in one click.
- Invoices can be created manually or auto-generated by the Automated Invoices engine.
- Products & Services is the central catalogue that feeds line items into Quotes, Invoices, and Automated Invoice configs.

---

## Page 1 — Quotes (`/billing/quotes`)

### Purpose
Create and send formal quotations to clients or prospects. Every field a professional South African quotation requires must be present. A quote can be converted to an invoice with one click.

### Data Model

```ts
Quote {
  id: uuid
  quote_number: string               // e.g. "QUO-2026-0011" — auto-generated, sequential
  client_id: uuid                    // FK → Client (existing client or prospect)
  contact_person: string | null      // name of person at client receiving the quote
  contact_email: string | null       // where the quote PDF is emailed
  status: enum                       // 'draft' | 'sent' | 'accepted' | 'declined' | 'expired' | 'converted'
  reference: string | null           // optional internal reference (printed on PDF header)
  issue_date: date
  expiry_date: date                  // default: issue_date + 30 days (configurable); printed as VALID UNTIL
  currency: string                   // default 'ZAR'
  sales_rep: string | null           // defaults to created_by user full name; printed on PDF header
  subject: string | null             // one-line description of what the quote is for
  scope_of_work: text | null         // rich text — detailed description of work/deliverables
  terms_and_conditions: text | null  // rich text — printed at bottom of PDF
  payment_terms: string | null       // e.g. "50% upfront, 50% on completion"
  notes: string | null               // visible to client, printed on PDF
  internal_notes: string | null      // staff-only, never printed
  discount_type: enum                // 'none' | 'percent' | 'fixed'
  discount_value: decimal            // applied at document level (after line totals)
  subtotal: decimal                  // sum of line totals before discount and tax
  discount_amount: decimal           // computed from discount_type + discount_value
  tax_total: decimal
  total: decimal
  converted_invoice_id: uuid | null  // populated when converted to invoice
  created_by: uuid                   // FK → User
  created_at: timestamp
  updated_at: timestamp
}

QuoteLineItem {
  id: uuid
  quote_id: uuid
  product_service_id: uuid | null    // null = fully custom line
  description: string                // editable even when pulled from catalogue
  quantity: decimal
  unit: string | null                // e.g. "hours", "devices", "months", "each"
  unit_price: decimal
  discount_percent: decimal          // line-level discount, default 0
  tax_rate: decimal                  // default 15 (VAT)
  line_total: decimal                // computed: qty × unit_price × (1 - discount) — tax added separately
  sort_order: int
}
```

### Quote PDF Template

Uses the **same base HTML template as the invoice** (`billing/templates/billing/invoice_pdf.html`) with the following differences applied via template context flags:

| Element | Invoice | Quote |
|---|---|---|
| Header label (top right) | `INVOICE` | `QUOTATION` |
| Header field label | `DATE` / `DUE DATE` | `DATE` / `VALID UNTIL` |
| Bank details block (bottom left) | Shown | Hidden |
| BALANCE DUE block | Shown | Hidden |
| Scope of work section | Hidden | Shown (if populated) |
| Payment terms line | Hidden | Shown (if populated) |
| Terms & conditions section | Hidden | Shown (if populated) |
| Footer note | Configurable | "This quotation is valid until [date]. To accept, reply to this email or contact us." |

The FROM/TO address block, line items table (same 7 columns), totals block, and logo/metadata header all render identically to the invoice. The quote totals block shows Total Discount, Total Exclusive, Total VAT, Sub Total, and Grand Total — but omits the BALANCE DUE line.

Scope of work (if populated) renders as a full-width section between the address block and the line items table.

### Quote → Invoice Conversion

```
User clicks "Convert to Invoice" on a quote (any status except 'void')
  └── Confirmation modal: "Convert QUO-2026-0011 to Invoice?"
  └── New Invoice created:
        source: 'converted_quote'
        quote_id: set
        issue_date: today
        due_date: today + client payment terms
        status: DRAFT
        all line items cloned from quote
  └── Quote status → 'converted'
  └── User redirected to the new Invoice detail page for review before sending
  └── Converted invoice link appears on the quote detail page
```

## Page 2 — Invoices (`/billing/invoices`)

### Purpose
View all invoices (auto-generated and manual), create new invoices from scratch or from a converted quote, track payment status, record payments, and send invoices to clients.

### Data Model

```ts
Invoice {
  id: uuid
  invoice_number: string             // e.g. "INV-2026-0042" — auto-generated, sequential
  client_id: uuid
  contact_email: string | null       // where the PDF is sent; defaults to client billing email
  status: enum                       // 'draft' | 'approved' | 'sent' | 'paid' | 'overdue' | 'partially_paid' | 'cancelled' | 'void'
  source: enum                       // 'manual' | 'converted_quote' | 'automated'
  quote_id: uuid | null              // set if converted from quote
  automated_invoice_config_id: uuid | null  // set if generated by automation engine
  reference: string | null           // optional internal reference number (printed on PDF header)
  issue_date: date
  due_date: date                     // default: issue_date + client payment terms (days); printed as DELIVERY DATE on PDF
  currency: string                   // default 'ZAR'
  sales_rep: string | null           // defaults to created_by user full name; printed on PDF header
  subject: string | null
  notes: string | null               // printed on invoice
  internal_notes: string | null      // not printed
  discount_type: enum                // 'none' | 'percent' | 'fixed'
  discount_value: decimal
  subtotal: decimal
  discount_amount: decimal
  tax_total: decimal
  total: decimal
  amount_paid: decimal               // updated as payments recorded
  balance_due: decimal               // computed: total - amount_paid
  sent_at: timestamp | null
  paid_at: timestamp | null
  created_by: uuid
  created_at: timestamp
  updated_at: timestamp
}

InvoiceLineItem {
  id: uuid
  invoice_id: uuid
  product_service_id: uuid | null
  description: string
  quantity: decimal
  unit: string | null
  unit_price: decimal
  discount_percent: decimal
  tax_rate: decimal
  line_total: decimal
  sort_order: int
}

Payment {
  id: uuid
  invoice_id: uuid
  amount: decimal
  payment_date: date
  method: enum                       // 'eft' | 'cash' | 'card' | 'other'
  reference: string | null
  notes: string | null
  created_by: uuid
  created_at: timestamp
}
```

Status transitions:
- `amount_paid >= total` → auto-set to `PAID`
- `amount_paid > 0 AND amount_paid < total` → auto-set to `PARTIALLY_PAID`
- `due_date < today AND status in ['sent', 'partially_paid']` → nightly job sets to `OVERDUE`

#### PDF rendering notes

- Use WeasyPrint with a Django HTML template
- The template file should be `billing/templates/billing/invoice_pdf.html`
- Same base template is used for quotes — the header label switches between `INVOICE` and `QUOTATION`, the delivery date row becomes `VALID UNTIL`, and the bank details block is hidden on quotes
- Logo is rendered as a base64-embedded image so the PDF is self-contained
- Page size: A4, portrait
- Font: clean sans-serif (e.g. system font stack or embedded Helvetica)
- The `BALANCE DUE` amount at the bottom right must be the largest, boldest text on the page — same visual weight as in the reference doc

If converted from a quote, show a small line beneath the header block: `This invoice was generated from Quote QUO-2026-0011`.

## Page 4 — Automated Invoices (`/billing/automated`)

### Purpose
Configure recurring invoice schedules per client. Each configuration defines which services to bill, when to send, whether it recurs every month indefinitely or for a fixed period, and supports one-off additions for a single month without breaking the regular schedule.

### Core Concept

An **Automated Invoice Config** is a standing instruction tied to a client:
> "On day X of every month, generate a draft invoice for [Client] containing [these line items]. Send it automatically (or as a draft for review). If there are any one-time additions for this billing cycle, include them too, then drop back to the base config next cycle."

### Data Model

```ts
AutomatedInvoiceConfig {
  id: uuid
  name: string                       // internal label, e.g. "Acme Corp — Monthly Retainer"
  client_id: uuid                    // FK → Client
  is_active: boolean
  recurrence: enum                   // 'monthly_indefinite' | 'monthly_fixed_term' | 'once'
  start_date: date                   // first generation date
  end_date: date | null              // null for indefinite; set for fixed-term or once-off
  day_of_month: int                  // 1–28 (day to generate; capped at 28 to avoid month-end issues)
  generation_mode: enum              // 'draft' (staff reviews before send) | 'auto_send' (send immediately)
  payment_terms_days: int            // default 30; sets due date on generated invoice
  subject_template: string           // supports {month}, {year}, {client_name} — e.g. "Monthly IT Services — {month} {year}"
  notes_template: text | null        // printed on each generated invoice; same variable support
  internal_notes: string | null
  next_run_date: date                // computed; updated after each generation run
  last_run_date: date | null
  created_by: uuid
  created_at: timestamp
  updated_at: timestamp
}

AutomatedInvoiceConfigLineItem {
  id: uuid
  config_id: uuid                    // FK → AutomatedInvoiceConfig
  product_service_id: uuid           // FK → ProductService
  custom_description: string | null  // overrides catalogue description if set
  custom_unit_price: decimal | null  // overrides catalogue price if set
  quantity: decimal                  // default 1
  sort_order: int
}

// One-time addition for a specific upcoming cycle
AutomatedInvoiceOneTimeAddition {
  id: uuid
  config_id: uuid                    // FK → AutomatedInvoiceConfig
  billing_month: date                // the month this addition applies to (stored as first-of-month)
  product_service_id: uuid | null    // null = fully custom line
  description: string
  quantity: decimal
  unit: string | null
  unit_price: decimal
  tax_rate: decimal
  notes: string | null               // internal only; why this was added
  status: enum                       // 'pending' (not yet billed) | 'billed' (included in generated invoice) | 'cancelled'
  created_by: uuid
  created_at: timestamp
}
```

### Generation Engine Logic

```
Celery Beat — runs daily at 06:00 SAST
  └── Find all active AutomatedInvoiceConfigs where next_run_date = today
        └── For each config:
              1. Pull base line items from AutomatedInvoiceConfigLineItem
              2. Pull any AutomatedInvoiceOneTimeAddition where
                    billing_month = this_month AND status = 'pending'
              3. Create Invoice:
                    source: 'automated'
                    automated_invoice_config_id: config.id
                    status: 'draft' (or auto-send if generation_mode = 'auto_send')
                    subject: rendered subject_template
                    line items: base items + one-time additions
              4. Mark all one-time additions used → status: 'billed'
              5. Update config:
                    last_run_date = today
                    next_run_date = compute_next(recurrence, day_of_month)
                    if recurrence = 'once' OR (recurrence = 'monthly_fixed_term' AND next_run_date > end_date):
                        is_active = false
              6. Notify staff: "Automated invoice generated for [Client]"
```

**Next run date computation:**
- `monthly_indefinite`: next month, same day
- `monthly_fixed_term`: next month, same day — if past `end_date`, deactivate
- `once`: no next run, deactivate after generation

**4. Generation History**
- Table of all invoices generated from this config: Invoice No | Period | Generated On | Status | [View Invoice link]

## Tech Notes for Implementation

### Backend (Django)

**App:** `billing`

**Models:**
- `ProductService`
- `Quote`, `QuoteLineItem`
- `Invoice`, `InvoiceLineItem`, `Payment` 
- `AutomatedInvoiceConfig`, `AutomatedInvoiceConfigLineItem`, `AutomatedInvoiceOneTimeAddition`
- `BillingSettings` (singleton model — one row per company)

**Tasks (Celery + Celery Beat):**
- `generate_automated_invoices` — daily at 06:00 SAST, finds all configs due today
- `mark_overdue_invoices` — daily, transitions sent/partially_paid past due_date to overdue
- `send_overdue_reminders` — daily, checks reminder schedule per client

**PDF generation:** `WeasyPrint` — Django HTML template → PDF binary → attach to email or serve as download. One base template, slight variation for invoice vs quote (header label, payment details section conditional).

**Numbering:** atomic `select_for_update()` on a `BillingCounter` model to guarantee no gaps.

**All monetary fields:** `DecimalField(max_digits=12, decimal_places=2)`
